# نظام التحديثات

يعتمد التطبيق على ملف `version.json` الموجود في جذر المشروع.

عند فتح التطبيق أو رجوع الإنترنت، يفحص التطبيق الملف بدون كاش:

```text
./version.json
```

إذا كان رقم النسخة في الملف أكبر من رقم النسخة داخل `src/core/config.js`، تظهر نافذة للمستخدم فيها:

- النسخة الحالية.
- النسخة الجديدة.
- تاريخ الإصدار.
- ملخص التحديث.
- قائمة التغييرات.
- أزرار: تحديث الآن / ذكرني لاحقًا / البقاء على النسخة الحالية.

## عند إصدار نسخة جديدة

1. غيّر `APP_CONFIG.version` في:

```text
src/core/config.js
```

2. غيّر `CACHE_NAME` و `RUNTIME_CACHE` في:

```text
sw.js
```

3. حدّث `version.json` بنفس رقم النسخة الجديدة وقائمة التغييرات.

4. ارفع الملفات إلى GitHub Pages.

5. عند فتح التطبيق، تظهر نافذة التحديث للمستخدم.

## ملاحظة مهمة

لو اختار المستخدم "البقاء على النسخة الحالية"، لن تظهر نفس النسخة مرة أخرى حتى يتم رفع رقم نسخة أحدث في `version.json`.
